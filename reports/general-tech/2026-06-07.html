◆ Your stack — web platform & dev tools

Feature Jun 4–5, 2026

Vercel ships a Shopify storefront builder, Sandbox persistent storage, and a skills API

Three changelog drops in your exact window. June 4: you can create a Shopify test store directly from Vercel, build a production storefront with v0, and claim ownership later. June 5: "Drives" — persistent, attachable storage with a lifecycle independent of any sandbox — entered private beta for Vercel Sandbox, and the new skills.sh API went live, letting you query 600,000+ open-source skills via Vercel OIDC tokens.

So what for TSD: The Shopify path is a direct upsell for any SMB client who wants real e-commerce without you hand-rolling a cart — build it with v0, hand the store over, keep the retainer. Sandbox Drives is worth a look for anything stateful you've been running in ephemeral containers.

Fix May 2026 release

Patch reminder — confirm you're on the patched Next.js / React minors

No fresh CVE landed June 3–7. But if you skipped the May 2026 coordinated Next.js release, that's the one to verify: 13 advisories including a high-severity WebSocket SSRF in self-hosted Next.js (CVE-2026-44578, CVSS 8.6) and middleware/proxy authorization bypasses. Fixed in Next.js 15.5.18 and 16.2.6 — earlier minors are not backported, so you must move to a patched minor. Separately, React 19.2.7 / 19.1.8 / 19.0.7 (June 1) fix missing FormData entries in Server Actions.

So what for TSD: Run npx next --version across every client repo today. Anything self-hosting Next below 15.5.18 or 16.2.6 needs the bump — the SSRF is the load-bearing one. The Server Actions FormData fix matters anywhere you handle form submissions.

Feature Jun 3, 2026

Deno 2.8.2 adds post-quantum crypto and deno compile --bundle

The latest Deno runtime adds ML-DSA and ML-KEM post-quantum cryptography, ChaCha20-Poly1305 and the SHA-3/SHAKE family, a new --bundle flag for deno compile, --env-file support in tasks, and a batch of Node.js-compatibility fixes (fs, HTTP/HTTPS proxying, NAPI).

So what for TSD: Not a switch-now item, but deno compile --bundle makes single-binary CLI tooling genuinely easy — handy if you ever ship a client a small standalone utility. Filing it as "know it exists."

◆ Chips & hardware

Breaking Jun 4–5, 2026

Semiconductor sell-off: AMD, Intel, Broadcom, Micron, ARM all sink

Chip stocks dropped sharply June 4–5, led by a cautious AI-chip outlook from Broadcom (record results, but no guidance raise), a deepening memory-chip crunch, and a projected slump in global smartphone demand. AMD and Intel led the declines June 5.

So what for TSD: Indirect, but real: sustained hardware-cost pressure eventually shows up in cloud and inference pricing. Nothing to act on — context for why your compute bills move.

New Computex, Jun 2–5

NVIDIA, Microsoft, and ARM tease NVIDIA's first consumer CPU in over a decade

Computex in Taipei centered on a coordinated NVIDIA/Microsoft/ARM announcement pointing at NVIDIA's first consumer CPU in more than ten years, alongside an RTX "Spark" superchip push into laptops and desktops. The signal: a real ARM-on-Windows consumer assault on the x86 Intel/AMD duopoly — context for the sell-off above.

So what for TSD: Long-horizon. If ARM-on-Windows gains share, "does it run on ARM" becomes a real QA dimension for desktop tooling. For web work it barely matters — browsers abstract it away.

Breaking Jun 2–7, 2026

DentaQuest breach exposes 2.6M people; ShinyHunters leaks 234GB

Extortion group ShinyHunters publicly leaked a claimed 234GB trove from DentaQuest, a major US dental/vision benefits administrator, after the company refused to pay — exposing PII and PHI for 2.6M accounts: names, DOBs, government IDs, Medicaid IDs, and insurance data. DentaQuest confirmed the incident June 2; the public leak landed in the June 4–7 window. The attack fits the year's dominant pattern: help-desk impersonation and MFA-reset social engineering.

So what for TSD: The lesson is the vector, not the victim. The 2026 breach pattern is social engineering against support staff, not zero-days. For any client with a login: enforce MFA, and make sure password/MFA resets can't be triggered by a phone call alone.
