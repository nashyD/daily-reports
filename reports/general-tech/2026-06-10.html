◆ Patch first

Critical Jun 9, 2026

Wormable Windows kernel RCE headlines a record-volume Patch Tuesday

Microsoft's June 2026 Patch Tuesday addressed on the order of 198 CVEs (32 critical, the rest important), one of its largest single releases on record across Windows, Office, Edge, Azure, .NET, Visual Studio, Exchange, Hy-V, Secure Boot, and BitLocker. The standout is CVE-2026-45657, a use-after-free in the Windows kernel's TCP/IP handling that allows unauthenticated remote code execution over the network with no user interaction — a CVSS 9.8 that analysts flag as wormable. It was not publicly disclosed or known to be exploited at publication, but the profile is exactly what gets weaponized fast.

Separately, attackers are actively exploiting CVE-2026-3300 in Everest Forms Pro, a WordPress plugin, via PHP eval() injection for unauthenticated RCE; and a Cisco SD-WAN 0-day (CVE-2026-20245) reported June 5 had no patch at disclosure.

So what for TSD: Your dev Mac and any Windows machine you touch should take the June updates now — a wormable kernel bug is the kind that spreads without anyone clicking anything. More relevant to client work: if any TSD or client site runs WordPress with Everest Forms, that one is being exploited in the wild right now — check and patch or pull the plugin today. Most TSD builds are Next.js, so the WordPress bugs likely don't touch you, but confirm rather than assume for inherited client sites.

◆ On TSD's own stack

Fix May 2026

Next.js coordinated security release: 13 advisories, patching is the only full fix

Vercel's May 2026 coordinated Next.js security release fixed 13 advisories spanning denial of service (including connection exhaustion in apps using Cache Components), middleware/proxy bypass in App Router apps via segment-prefetch and dynamic route-parameter injection, server-side request forgery via WebSocket upgrades, cache poisoning, and XSS. One tracks an upstream React Server Components issue (CVE-2026-23870). Vercel stated it did not ship WAF rules for this batch — these cannot be reliably blocked at the edge, so upgrading is the only complete mitigation. A June 16.2 backport followed with stability and bug fixes (hydration, router, cache-tag encoding, server actions, Turbopack, FormData).

So what for TSD: Direct action item. Audit every live TSD and client Next.js app — tsd-modernization-app, the State Farm renters funnel, Bisque/Clay, AB Carter, Cake Me Away, Praxis — and confirm each is on a patched Next.js. The middleware/proxy-bypass advisories matter most for any app gating routes (auth, the client portal, the sales dashboard). Since these can't be WAF'd, a redeploy on a patched version is the fix. If you patched in May, just verify; if any app has been quiet since, bump it.

Migrate 2026

Supabase is deprecating legacy API keys — move to publishable / secret keys

Supabase's legacy anon and service_role API keys are being deprecated by the end of 2026 in favor of the new publishable (sb_publishable_…) and secret (sb_secret_…) key format. It is a planned migration, not an emergency, but every Supabase-backed app needs the swap before the deadline. Vercel's Supabase integration has also tightened: multiple Vercel projects can connect to one Supabase project, and auth redirect URIs now auto-sync to your main and preview deployment domains via deployment webhooks.

So what for TSD: Put the key migration on the calendar now rather than getting surprised by a breakage in Q4. Any TSD app on Supabase (the portal, the sales dashboard) should plan to rotate to the new key format and update Vercel env vars. The auto-syncing redirect URIs are a quiet win — fewer manual auth-URL fixes when you spin up preview deploys. (You already learned the RP-origin lesson on the portal passkeys; same discipline applies here.)

◆ Hardware

New Early Jun 2026

Nvidia enters the PC chip market with the Arm-based RTX Spark

Nvidia is making its first move into consumer PC processors with the RTX Spark, an Arm-based superchip pairing a Blackwell RTX GPU with a Grace CPU, unveiled by Jensen Huang at Computex. It debuts this fall in Windows laptops from Microsoft, Dell, HP, ASUS, Lenovo, and MSI, taking direct aim at Apple Silicon and Qualcomm in the Arm-PC segment. Related: Apple's overhauled Gemini-powered Siri will reportedly run on Nvidia Blackwell B200 chips accessed via Google Cloud.

So what for TSD: No immediate impact on what you ship, but the Arm-PC shift is worth tracking for two reasons: more capable on-device compute eventually means more local AI inference (cheaper concierge/receptionist features that don't hit an API), and a new dominant laptop architecture changes how you test client sites for performance. Nothing to act on; just know the desktop is moving to Arm.
