◆ The model

New 2.1.170

Claude Fable 5 is generally available — update to get it

Anthropic shipped Claude Fable 5, a "Mythos-class" model it has made safe for general use. The changelog's claim is direct: "Fable's capabilities exceed those of any model we've ever made generally available." Access is gated on the client — you need Claude Code 2.1.170 to select it.

No benchmark numbers or pricing are in the changelog; the linked announcement is the place to confirm cost and limits before you lean on it. Treat the "most capable ever" line as Anthropic's framing until you've run it on your own work.

So what for TSD: You're running Opus 4.8 day to day and you've already been handing the hairier Praxis Field and roadmap work to Fable 5. Update to 2.1.170 so that handoff is a model switch inside one client instead of context-juggling. Before you make it your default driver, check the announcement for the price-per-token — a "most capable ever" model is the kind of thing you reach for on a gnarly Bisque/Praxis problem, not on every routine edit. Run it head-to-head on one real task this week and decide where it earns its cost.

◆ The fix that could have cost you history

Fix 2.1.170

VS Code terminal sessions were silently dropping their transcripts

Sessions started from the VS Code integrated terminal — or any shell that inherited Claude Code's environment variables — weren't saving transcripts, and so never showed up under --resume. 2.1.170 fixes it.

This is the kind of bug you don't notice until you go looking for a session that should be there and isn't.

So what for TSD: You launch Praxis, Praxis Field, and the TSD apps straight from VS Code (your launch.json configs). If you've been on a build between the regression and now, some of those sessions may not be resumable. Update, and don't count on --resume recovering anything from before the patch — if there's a session you were planning to pick back up, check it exists now rather than assuming.
